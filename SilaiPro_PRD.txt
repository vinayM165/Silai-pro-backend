1. Executive Summary
SilaiPro ek complete digital management solution hai jo tailors, boutique owners, aur unke staff ke liye design kiya gaya hai. Ye system customer measurements ko digitally store karta hai, orders ko track karta hai, billing aur accounts manage karta hai, aur staff ko role-based access deta hai.


Ye app Flutter framework mein build hogi jo Android phones (portrait) aur tablets (landscape) dono pe achhi tarah se kaam kare. Backend Java Spring Boot REST APIs use karega jo secure, scalable, aur maintainable ho.

2. Problem Statement & Goals
2.1 Current Challenges
Aaj bhi bahut se tailors aur boutique owners apne records paper mein ya simple registers mein rakhte hain. Isse kai problems hoti hain:


* Customer ke naap (measurements) dhundhne mein time lagta hai ya khoj jaate hain
* Bills manually likhne padte hain, galtiyan hoti hain
* Partial payment ka koi systematic record nahi hota
* Staff ko pata nahi chalta ke kaunsa order kab deliver hona hai
* Customer ki purani history easily available nahi hoti
* Business ke accounts ka clear picture nahi milta


2.2 Solution Goals
* Ek centralized digital system banana jo sab kuch ek jagah rakh sake
* Role-based access se security ensure karna
* Customer measurements dynamic aur configurable rakhna
* Billing aur partial payment tracking automate karna
* Print aur messaging se customer experience improve karna
* Staff productivity aur accountability badhana

5. Measurements Module (Dynamic Naap System)
5.1 Dynamic Fields – Backend Se Configure
Ye system ka ek important feature hai. Measurement fields hard-coded nahi hain – Admin backend se apni zaroorat ke hisaab se fields add, edit, ya delete kar sakta hai.


* Admin login karke 'Measurement Templates' section mein jaata hai
* Wahan naap ke categories bana sakta hai – jaise 'Gents Suit', 'Ladies Salwar', 'Sherwani', 'Kids', etc.
* Har category ke andar custom fields add kar sakta hai – jaise 'Length', 'Chest', 'Waist', 'Hip', 'Sleeve', 'Collar', etc.
* Field type choose kar sakta hai: Number (cm/inch), Text, Select (dropdown), Notes
* Field order drag-drop se change kar sakta hai
* Optional ya required mark kar sakta hai
* Unit preference set kar sakta hai – CM ya Inch (globally ya per customer)

6.2 Order Status Workflow
Har order ek defined workflow follow karta hai:


Order Status Flow
	1. ORDER RECEIVED  →  Order create hoti hai, advance liya
	2. IN CUTTING      →  Fabric cut ho raha hai
	3. IN STITCHING    →  Silai chal rahi hai
	4. TRIAL PENDING   →  Customer ko trial ke liye bulana hai
	5. ALTERATION      →  Trial ke baad changes
	6. READY           →  Delivery ke liye taiyar hai
	7. DELIVERED       →  Customer ko de diya, final payment
	8. CANCELLED       →  Order cancel
	

6.3 Order Dashboard

* Overdue orders – jinki delivery date nikal gayi
* Status-wise grouping – kitne orders kahan hain
* Quick status update – staff swipe ya tap se status change kare
* Order search by customer name, order ID, date, status

7. Billing & Invoice Module
7.1 Invoice Generation
Har order ke liye ek proper invoice ban sakti hai:


* Invoice number auto-generate (INV-2025-0001)
* Customer ki details automatically pull hoti hain
* All order items with quantity, rate, amount list
* Subtotal, discount, total amount
* GST/Tax option (optional – enable/disable from settings)
* Advance paid deduct karke balance due dikhayega
* Invoice date aur due date
* Payment terms / notes section


7.2 Partial Payment Tracking
Ye ek core feature hai. Agar kisi ka total bill Rs. 20,000 hai aur usne Rs. 10,000 diye hain:


Payment Record Field
	Description
	Invoice Total
	Rs. 20,000 (total bill)
	Payment 1
	Rs. 10,000 – Date: 15 Jan – Mode: Cash – Note: Advance
	Remaining Balance
	Rs. 10,000 (auto-calculated)
	Payment 2
	Rs. 5,000 – Date: 20 Jan – Mode: UPI – Note: Partial
	Remaining Balance
	Rs. 5,000 (live updated)
	Payment Status
	Partial / Paid / Overdue (auto-update)
	Reminder Flag
	Manually mark karo – remind karna hai
	

Har payment record mein:
* Amount paid
* Date of payment
* Payment mode: Cash, UPI, Bank Transfer, Cheque (manual entry)
* Staff ka naam jo payment liya
* Optional receipt number
* Notes/remarks


7.3 Print Invoice
* Professional PDF invoice generate karo
* Shop logo, name, address, GST number include
* Customer ke details, order items, payment history sab
* Thermal printer (58mm/80mm) ke liye compressed layout
* A4 print ke liye full layout
* Share via WhatsApp ya email directly app se

12. Security & Data Protection
12.1 Authentication & Authorization
* JWT token based authentication – access token (1 hour) + refresh token (30 days)
* Role permissions check both frontend (UI hide) aur backend (API reject) pe
* PIN-based quick login for staff on shared devices
* Auto logout after configurable idle time
* Failed login attempts limit (5 attempts → 30 min lockout)


12.2 Data Security
* All API calls HTTPS only (SSL/TLS)
* Passwords bcrypt hash se store honge
* Sensitive data (payment amounts) encrypted at rest
* Database backups – daily automated backup
* Audit log – koi bhi delete ya payment change kare to log maintain ho
